## 修改计划

### 问题分析
当前 `get_class_students` 函数的 SQL 查询存在以下问题：
1. **字段不存在**：
   - 论文表 (`papers`) 中没有 `title` 字段
   - 论文版本表 (`paper_versions`) 中没有 `is_latest` 字段

2. **表结构不匹配**：
   - 论文表结构：`id`, `owner_id`, `latest_version`, `ddl`, `oss_key`, `created_at`, `updated_at`
   - 论文版本表结构：`id`, `paper_id`, `teacher_id`, `version`, `size`, `created_at`, `updated_at`, `status`, `submitted_by_id`, `submitted_by_name`, `submitted_by_role`

### 解决方案
修改 `app/api/v1/endpoints/groups.py` 文件中的 `get_class_students` 函数：

1. **更新 SQL 查询**：
   - 移除对 `p.title` 的引用
   - 移除对 `pv.is_latest` 的引用，改为获取最新版本的论文
   - 调整查询逻辑，使用正确的表结构

2. **修改结果处理**：
   - 移除 `paper_title` 字段，因为数据库中不存在
   - 调整版本号处理逻辑
   - 确保状态字段正确映射

3. **保持功能完整性**：
   - 仍然返回学生基本信息
   - 仍然返回学生的论文列表
   - 仍然返回论文的版本、状态、更新时间和批注数

### 具体修改步骤
1. 修改 SQL 查询语句，使用正确的字段和表结构
2. 更新结果处理逻辑，适配修改后的查询结果
3. 测试接口是否正常工作

### 预期结果
- 接口不再返回 500 错误
- 正确返回班级学生列表及其论文状态
- 所有数据都来自现有的数据库表，不添加新表
